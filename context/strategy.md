# Strategie

## Aktueller Fokus

Prototyp fertigstellen und erste Feldtests im eigenen Arbeitsumfeld durchfuehren.

## Strategische Prioritaeten

### 1. RS232-Kommunikation knacken

- WD290-ECU Protokoll per Reverse Engineering entschluesseln
- Win7-VM mit ECU Cadi + Serial Sniffer ist eingerichtet
- Ziel: Sterilisator-Chargenprotokolle live mitlesen

### 2. Software stabilisieren

- PDF-Generator, Dashboard, Watchdog produktionsreif machen
- Patch-Dateien konsolidieren (15+ separate Patches)
- Tests ausfuellen und automatisieren

### 3. Hardware-Prototyp testen

- Raspberry Pi mit Sensoren im Feld einsetzen
- Druck, Strom, digitale Signale erfassen
- Ergebnisse dokumentieren

### 4. Langfristig: CE-konforme Hardware

- Migration auf Unipi Neuron oder RevPi Connect
- Regulatorik klaeren (kein eigenes Medizinprodukt, nur Diagnosetool)

## Wie Erfolg aussieht

- Funktionierender Prototyp, der im Feld einsetzbar ist
- RS232-Protokoll entschluesselt und implementiert
- Erste Feldtests erfolgreich durchgefuehrt
